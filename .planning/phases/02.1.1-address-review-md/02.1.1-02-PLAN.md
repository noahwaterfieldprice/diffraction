2. **Fix PointGroup bare KeyError → ValueError with suggestions (M3/R4):**
   - Line ~108 currently: `number = POINT_GROUP_NUMBERS[symbol]` — bare dict lookup
   - Replace with guarded lookup using `difflib.get_close_matches`:
   ```python
   import difflib

   try:
       number = POINT_GROUP_NUMBERS[symbol]
   except KeyError:
       close = difflib.get_close_matches(
           symbol, POINT_GROUP_NUMBERS.keys(), n=3, cutoff=0.6
       )
       suggestion = (
           f" Did you mean: {', '.join(repr(m) for m in close)}?"
           if close
           else ""
       )
       raise ValueError(
           f"Unknown point group symbol {symbol!r}.{suggestion}"
       ) from None
   ```

3. **Convert PointGroup to `@dataclass(frozen=True, init=False)`:**
   - Add `from dataclasses import dataclass, field` to imports
   - Change class declaration to `@dataclass(frozen=True, init=False)`
   - Declare fields:
     - `number: int`
     - `symbol: str`
     - `operators: dict[str, Matrix] = field(repr=False)`
   - Keep the existing `__init__` method. Since `init=False`, the dataclass does NOT generate an `__init__`. The existing `__init__` must use `object.__setattr__` to set fields because `frozen=True` prevents normal assignment:
     ```python
     def __init__(self, *, number: int = 0, symbol: str = "") -> None:
         # ... existing logic to determine number from symbol or vice versa ...
         data = self._load_point_group_data(resolved_number)
         object.__setattr__(self, "number", resolved_number)
         object.__setattr__(self, "symbol", data["symbol"])
         object.__setattr__(self, "operators", data["operators"])
     ```
   - **Remove the manual `__repr__`** — the dataclass-generated `__repr__` will show `PointGroup(number=X, symbol='Y')` (operators excluded via `repr=False`). The old repr was `PointGroup(number=X, symbol='Y')` so this matches.
   - Keep `_load_point_group_data` as a `@staticmethod` — it does file I/O and returns parsed data
   - `frozen=True` prevents re-assignment of `number`, `symbol`, `operators` after init. The `operators` dict itself can still be mutated (dict contents), but the attribute reference is immutable. This is correct behavior.
